# CONVEX KNOWLEDGE BASE

## OVERVIEW

Convex TypeScript realtime backend with Clerk auth and reactive queries/mutations.

## STRUCTURE

```
convex/
  schema.ts          # Table definitions, indexes, validators
  auth.config.ts     # Clerk JWT provider configuration
  http.ts            # HTTP actions (webhooks, health)
  users.ts           # User queries/mutations (syncFromClerk, etc.)
  recipes.ts         # Recipe queries/mutations
  handle.ts          # HTTP action handlers
  handle.test.ts     # Co-located tests (vitest)
  example.ts         # Example functions (can be removed)
  _generated/        # Auto-generated by Convex (do not edit)
    server.ts        # query, mutation, action exports
```

## WHERE TO LOOK

| Task               | Location                | Notes                                           |
| ------------------ | ----------------------- | ----------------------------------------------- |
| Add table          | `convex/schema.ts`      | Use `defineTable`, add indexes for queries      |
| Add query/mutation | `convex/*.ts`           | Export with `query({...})` or `mutation({...})` |
| Auth integration   | `convex/auth.config.ts` | Clerk JWT issuer domain from env                |
| HTTP webhook       | `convex/http.ts`        | `httpAction` with `httpRouter()`                |
| Test function      | `convex/*.test.ts`      | Vitest, co-located with source                  |

## CONVENTIONS

**Functions**

- Import from `./_generated/server`: `query`, `mutation`, `action`
- Use `ctx.auth.getUserIdentity()` for Clerk auth
- Use `ctx.db.query("table").withIndex("index")` for indexed lookups
- Throw errors for auth failures; Convex handles HTTP 500

**Schema**

- Use `v` validators from `convex/values`
- Define indexes for every query pattern: `.index("by_field", ["field"])`
- Use `v.optional()` for nullable fields

**Auth**

- Clerk JWT configured in `auth.config.ts` via `CLERK_JWT_ISSUER_DOMAIN`
- `identity.subject` is the Clerk user ID (stored as `clerkId`)
- Call `users.syncFromClerk` after mobile sign-in to sync user record

**Tests**

- Co-located `*.test.ts` files, run with `vitest`
- Mock Convex context or test against dev deployment

## ANTI-PATTERNS

- Never edit `convex/_generated/` — run `convex dev` to regenerate
- Do not duplicate auth checks in FastAPI and Convex — Convex owns auth for its functions
- Avoid unindexed queries in production — always add indexes for `withIndex` calls
- Do not call Convex from server-side FastAPI — use FastAPI for heavy compute, Convex for realtime state
