# Documentation standards

Documentation in this repo follows the **documentation guide** for structure, style, and maintenance.

- **Full guide:** [.agents/skills/docs-write/references/documentation-guide.md](../.agents/skills/docs-write/references/documentation-guide.md) — principles, Divio doc types, directory structure, visuals (Mermaid/PlantUML), API docs, ADRs and runbooks, when to document, checklists.

**TL;DR:** Keep docs focused and visual; use active voice and consistent terminology; organize by topic (guides = how-to, reference = facts, architecture/ADR = explanation); update docs when code changes. Store all docs under `.docs/`.

**Index:** [.docs/README.md](README.md) — start here for the full doc index and “Start here” links.
