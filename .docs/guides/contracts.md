# Contracts and codegen

When to regenerate API types and how CI keeps them in sync. Run `pnpm contracts:generate` from the repo root after you change FastAPI routes or Pydantic models.

- [When to run](#when-to-run) · [CI and drift](#ci-and-drift) · [Do not edit generated files](#do-not-edit-generated-files)

## When to run

Run **`pnpm contracts:generate`** from the repo root after you change FastAPI routes or Pydantic models in `apps/api`. The command exports the OpenAPI schema from the API and regenerates TypeScript types in `packages/contracts`; consumers (e.g. `@korb/mobile`) get updated types from `@korb/contracts`.

Typical flow: edit `apps/api/src/routes/*.py` or response/request models → run `pnpm contracts:generate` → commit the updated generated files if you intend to use the new types in the monorepo.

## CI and drift

CI runs `pnpm contracts:generate` and checks for **drift**: if the generated output differs from what’s committed, the pipeline fails. That keeps `packages/contracts/src/generated/` in sync with the FastAPI OpenAPI schema. Always run `pnpm contracts:generate` and commit any changes before pushing when you’ve changed the API surface.

## Do not edit generated files

Files under **`packages/contracts/src/generated/`** are generated by the contracts pipeline. Do not edit them by hand; any changes will be overwritten on the next `pnpm contracts:generate`.

**Generated file locations:**

- `packages/contracts/src/generated/index.ts` — package entry point
- `packages/contracts/src/generated/api/index.ts` — API client exports
- `packages/contracts/src/generated/api/models/*.ts` — Pydantic model types

**To change API types:** Update the FastAPI routes and Pydantic models in `apps/api/src/routes/`, then regenerate. Never hand-edit generated files.

## Contract pipeline

The full flow from FastAPI changes to committed TypeScript types:

1. **Edit FastAPI** — Modify Pydantic models or routes in `apps/api/src/routes/*.py`
2. **Regenerate** — Run `pnpm contracts:generate` from the repo root
3. **Review changes** — Check `git diff` for expected updates in `packages/contracts/src/generated/`
4. **Commit** — Include both `apps/api/openapi.json` and `packages/contracts/src/generated/**` changes

CI enforces this by running `pnpm contracts:generate` and failing if the output differs from committed files.
